# GitHub Actions Configuration for OpenClaw

This document describes the GitHub Actions workflows configured for the AI QA Agent with OpenClaw orchestration.

## Workflows

### 1. Main Pipeline: `ai-qa-agent.yml`

**Triggers:**
- `push` to branches: main, master, develop
- `pull_request` to branches: main, master, develop
- **Manual trigger** (workflow_dispatch) with customizable inputs

**Key Features:**
- Supports REST, GraphQL, and gRPC protocols
- Configurable orchestrator selection (openclaw, legacy)
- Debug logging option
- Multi-protocol test matrix
- GitHub Pages deployment

**Manual Run Inputs:**
```yaml
protocol:      # rest, graphql, grpc (default: rest)
spec_url:      # OpenAPI/GraphQL endpoint (default: examples/openapi.json)
api_base_url:  # API base URL (default: from secrets.API_BASE_URL)
orchestrator:  # openclaw or legacy (default: openclaw)
debug:         # true/false (default: false)
```

**Jobs:**
1. **run-ai-agent**: Main pipeline execution
   - Installs dependencies
   - Runs integration tests
   - Executes OpenClaw orchestrator
   - Generates Allure reports
   - Deploys to GitHub Pages

2. **validate-openclaw**: Validates OpenClaw integration
   - Verifies orchestrator import
   - Loads all workflow types
   - Checks documentation

3. **test-all-protocols**: Matrix testing
   - Tests each protocol (rest, graphql, grpc)
   - Tests each orchestrator (openclaw, legacy)
   - Generates per-protocol artifacts

### 2. OpenClaw Tests: `openclaw-tests.yml`

**Triggers:**
- `push` changes to `src/orchestrator/**` or `tests/**`
- `pull_request` to branches: main, master, develop
- Manual trigger (workflow_dispatch)

**Key Features:**
- Unit tests for orchestrator
- Example script validation
- Integration testing
- Workflow definition validation
- Documentation completeness check
- Performance baseline check
- Code quality checks

**Jobs:**
1. **openclaw-unit-tests**: RunUnit tests with coverage
2. **openclaw-examples**: Execute example scripts
3. **openclaw-integration**: Integration testing
4. **openclaw-workflow-validation**: Validate workflow definitions
5. **openclaw-documentation-check**: Verify documentation
6. **openclaw-performance-check**: Performance baseline
7. **openclaw-lint-check**: Code quality analysis
8. **summary**: Test results summary

## Usage

### Run Default Pipeline

```bash
# Automatically runs on push to main/master/develop
# Or manually trigger from GitHub UI
```

### Manual Trigger with Custom Inputs

**Via GitHub UI:**
1. Go to Actions → AI QA Agent with OpenClaw
2. Click "Run workflow"
3. Configure inputs:
   - Protocol: rest/graphql/grpc
   - Spec URL: path or URL to spec
   - Base URL: API endpoint
   - Orchestrator: openclaw or legacy
   - Debug: true or false
4. Click "Run workflow"

**Via GitHub CLI:**
```bash
gh workflow run ai-qa-agent.yml \
  -f protocol=rest \
  -f spec_url=examples/openapi.json \
  -f api_base_url=https://petstore.swagger.io/v2 \
  -f orchestrator=openclaw \
  -f debug=false
```


## Environment Variables

### Available in Workflows

```yaml
PROTOCOL:       # API protocol: rest, graphql, grpc
ORCHESTRATOR:   # Orchestrator: openclaw, legacy
LOG_LEVEL:      # Logging: DEBUG, INFO, WARNING, ERROR
DEBUG:          # Debug mode flag
REPORTS_DIR:    # Output directory for test reports
TESTS_DIR:      # Output directory for generated tests
```

### Secrets Required

```yaml
GEMINI_API_KEY:   # Google Gemini API key
API_BASE_URL:     # Default API base URL (optional)
```

**Set secrets:**
1. Go to Settings → Secrets and variables → Actions
2. Click "New repository secret"
3. Add `GEMINI_API_KEY` with your key
4. (Optional) Add `API_BASE_URL` for default endpoint

## Artifacts

### Generated Artifacts

All artifacts are retained for 30 days.

**Allure Report:**
- `allure-report-{protocol}-{orchestrator}`
- Contains HTML test report
- Deployed to GitHub Pages on master branch

**Test Results:**
- `test-results-{protocol}-{orchestrator}`
- Contains raw test data and generated test files
- Useful for debugging

**Example:**
- `test-results-rest-openclaw`
- `test-results-graphql-legacy`
- `test-results-grpc-openclaw`

### Accessing Artifacts

1. Go to Actions → Workflow run
2. Scroll to "Artifacts" section
3. Download desired artifact
4. For Allure report: Open `index.html` in browser

## Troubleshooting

### Workflow Fails

1. **Check logs:**
   - Go to Actions → Workflow run
   - Click failed job
   - Expand failing step
   - Check error messages

2. **Common issues:**
   - Missing GEMINI_API_KEY: Check secrets configuration
   - API unreachable: Verify base URL
   - grpcurl not found: Fails gracefully, continues with other tests

3. **Debug mode:**
   - Re-run with `debug=true` for verbose logging
   - Check "Run logs" section for detailed output

### Artifacts Not Generated

- Tests may fail silently (continue-on-error: true)
- Check "Generate Allure Report" step for errors
- Verify report directory has content

### GitHub Pages Not Updating

- Ensure running on master branch
- Verify Pages deployment job succeeded
- May take 1-2 minutes to update after deployment

## Performance Considerations

### Job Duration

- **run-ai-agent**: 5-15 minutes (depending on protocol)
- **validate-openclaw**: < 1 minute
- **openclaw-unit-tests**: 2-5 minutes
- **openclaw-examples**: 1-3 minutes
- **test-all-protocols**: 30-60 minutes (matrix)

### Resource Usage

- **Runners:** ubuntu-latest
- **Python:** 3.11
- **Timeout:** 60 minutes per job (run-ai-agent has 60 min timeout)

### Cost Optimization

- Use manual triggers for non-critical tests
- Limit matrix combinations
- Skip tests on documentation-only changes:

```yaml
# In workflow file:
paths-ignore:
  - '**.md'
  - 'docs/**'
```

## Best Practices

### 1. Use Manual Triggers for Expensive Tests

```yaml
# Avoid running matrix tests on every push
# Use workflow_dispatch for comprehensive testing
```

### 2. Monitor Action Logs

- Check run logs for failures
- Download artifacts for analysis
- Use debug mode for troubleshooting

### 3. Keep Secrets Secure

- Never commit GEMINI_API_KEY
- Rotate keys regularly
- Use organization secrets for shared access

### 4. Set Status Checks

- Require key workflows to pass before merge
- Prevents broken code from being merged

### 5. Regular Maintenance

- Monitor Github Actions usage
- Update action versions periodically
- Archive old artifacts

## Advanced Configuration

### Add New Protocol Test

Edit `ai-qa-agent.yml`, add to matrix:

```yaml
test-all-protocols:
  strategy:
    matrix:
      protocol: [rest, graphql, grpc, newprotocol]
      orchestrator: [openclaw, legacy]
```

### Add New Orchestrator

Edit `ai-qa-agent.yml`:

```yaml
test-all-protocols:
  strategy:
    matrix:
      protocol: [rest, graphql, grpc]
      orchestrator: [openclaw, legacy, neworchestr]
```

### Custom Deployment

Edit `ai-qa-agent.yml` deploy section:

```yaml
- name: Custom Deployment
  if: github.ref == 'refs/heads/main'
  run: |
    # Add custom deployment logic
```
