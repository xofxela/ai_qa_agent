# GitHub Actions Examples

This file contains examples and recipes for working with the GitHub Actions workflows.

## Quick Examples

### Example 1: Test REST API with OpenClaw (Default)

**Trigger via GitHub CLI:**
```bash
gh workflow run ai-qa-agent.yml \
  -f protocol=rest \
  -f spec_url=examples/openapi.json \
  -f api_base_url=https://petstore.swagger.io/v2 \
  -f orchestrator=openclaw
```

**What happens:**
- Installs dependencies
- Runs OpenClaw orchestrator for REST API
- Generates test file
- Runs tests with pytest
- Creates Allure report
- Uploads artifacts

**Duration:** 5-10 minutes

### Example 2: Test GraphQL with Debug Logging

```bash
gh workflow run ai-qa-agent.yml \
  -f protocol=graphql \
  -f spec_url=https://spacex-production.up.railway.app/ \
  -f api_base_url=https://spacex-production.up.railway.app/ \
  -f orchestrator=openclaw \
  -f debug=true
```

**What happens:**
- Enables DEBUG level logging
- Runs all steps with verbose output
- Useful for troubleshooting

**Logs:** Check Actions UI for detailed output

### Example 3: Test gRPC Protocol

```bash
gh workflow run ai-qa-agent.yml \
  -f protocol=grpc \
  -f api_base_url=grpcb.in:9000 \
  -f orchestrator=openclaw
```

**What happens:**
- Installs grpcurl
- Runs gRPC tests
- Note: spec_url is ignored for gRPC

### Example 4: Compare Orchestrators

```bash
# Test with OpenClaw
gh workflow run ai-qa-agent.yml \
  -f protocol=rest \
  -f orchestrator=openclaw

# Test with Legacy Agent
gh workflow run ai-qa-agent.yml \
  -f protocol=rest \
  -f orchestrator=legacy
```

**Compare:**
- Check both Allure reports
- Review execution times
- Compare test coverage

### Example 5: Test All Protocols

```bash
# Manually trigger matrix job
# This tests all combinations:
# - Protocols: rest, graphql, grpc
# - Orchestrators: openclaw, legacy
# Total: 6 parallel jobs
```

**Via GitHub UI:**
1. Go to Actions
2. Select "AI QA Agent with OpenClaw"
3. Click "Run workflow"
4. Leave defaults or configure
5. Click "Run workflow"

**Via CLI (all combinations):**
```bash
for protocol in rest graphql grpc; do
  for orchestrator in openclaw legacy; do
    gh workflow run ai-qa-agent.yml \
      -f protocol=$protocol \
      -f orchestrator=$orchestrator
  done
done
```

## Workflow Files Reference

### Main Workflow: ai-qa-agent.yml

Located: `.github/workflows/ai-qa-agent.yml`

**Jobs:**
1. `run-ai-agent`: Main testing job
2. `validate-openclaw`: Verify orchestrator
3. `test-all-protocols`: Matrix testing

**Triggers:**
- Push to main/master/develop
- Pull requests
- Manual (workflow_dispatch)

### OpenClaw Tests: openclaw-tests.yml

Located: `.github/workflows/openclaw-tests.yml`

**Jobs:**
1. `openclaw-unit-tests`: Run unit tests
2. `openclaw-examples`: Execute examples
3. `openclaw-integration`: Integration tests
4. `openclaw-workflow-validation`: Validate workflows
5. `openclaw-documentation-check`: Check docs
6. `openclaw-performance-check`: Performance test
7. `openclaw-lint-check`: Code quality
8. `summary`: Results summary

**Triggers:**
- Changes to src/orchestrator/**
- Changes to tests/**
- Manual trigger

### CI/CD Pipeline: ci-cd.yml

Located: `.github/workflows/ci-cd.yml`

**Jobs:**
1. `lint`: Linting and formatting
2. `test`: Run tests
3. `openclaw-validation`: Validate OpenClaw
4. `build`: Build check
5. `security`: Security scan
6. `quality-gate`: Gate check

**Triggers:**
- Push to main/master/develop
- Pull requests

## Accessing Results

### View Workflow Runs

**Via GitHub UI:**
1. Repository → Actions tab
2. Select workflow
3. Click run
4. View job logs and artifacts

**Via GitHub CLI:**
```bash
# List recent runs
gh run list

# View specific run
gh run view {run_id}

# View logs
gh run view {run_id} --log
```

### Download Artifacts

**Via GitHub UI:**
1. Go to workflow run
2. Scroll to "Artifacts"
3. Click artifact name
4. Download

**Via GitHub CLI:**
```bash
gh run download {run_id} -n {artifact_name}
```

**Example:**
```bash
# List artifacts
gh run view 12345

# Download Allure report
gh run download 12345 -n "allure-report-rest-openclaw"

# Extract and view
unzip allure-report-rest-openclaw.zip
cd allure-report
python -m http.server  # View index.html
```

### View GitHub Pages

```bash
# Report published to:
https://{username}.github.io/{repo}/

# Example:
https://xofxela.github.io/ai_qa_agent/
```

## Advanced Recipes

### Recipe 1: Scheduled Testing

**File:** `.github/workflows/scheduled-tests.yml`

```yaml
name: Scheduled Tests
on:
  schedule:
    - cron: '0 2 * * *'  # Daily at 2 AM UTC

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: pip install -r requirements.txt
      - run: |
          python -m src.main \
            --protocol rest \
            --spec examples/openapi.json \
            --base-url https://petstore.swagger.io/v2
```

**Usage:**
```bash
# Create file and commit
cp examples/scheduled-tests.yml .github/workflows/
git add .github/workflows/scheduled-tests.yml
git commit -m "Add scheduled tests"
git push
```

### Recipe 2: Slack Notifications

**Add to workflow:**
```yaml
- name: Notify Slack on Success
  if: success()
  uses: slackapi/slack-github-action@v1
  with:
    webhook-url: ${{ secrets.SLACK_WEBHOOK }}
    payload: |
      {
        "text": "✓ Tests passed for ${{ github.ref }}"
      }

- name: Notify Slack on Failure
  if: failure()
  uses: slackapi/slack-github-action@v1
  with:
    webhook-url: ${{ secrets.SLACK_WEBHOOK }}
    payload: |
      {
        "text": "✗ Tests failed for ${{ github.ref }}"
      }
```

**Setup:**
1. Create Slack webhook: Slack → App → Incoming Webhooks
2. Settings → Secrets → Add `SLACK_WEBHOOK`

### Recipe 3: Conditional Testing

**Run only on PR:**
```yaml
jobs:
  test:
    if: github.event_name == 'pull_request'
    runs-on: ubuntu-latest
    steps: ...
```

**Run only on specific paths:**
```yaml
on:
  push:
    paths:
      - 'src/**'
      - 'tests/**'
      - '.github/workflows/**'
```

**Skip testing:**
```bash
# In commit message
git commit -m "Update docs [skip ci]"
```

### Recipe 4: Deploy to Custom Domain

**Add to workflow:**
```yaml
- name: Deploy to Custom Domain
  if: github.ref == 'refs/heads/master'
  run: |
    # Deploy to your server
    deploy-script.sh ${{ secrets.DEPLOY_HOST }} allure-report
```

## Troubleshooting

### Workflow Not Triggering

**Check:**
1. Branch name matches trigger (main, master, develop)
2. File path correct: `.github/workflows/`
3. Syntax valid: Use GitHub workflow validator

**Fix:**
```bash
# Verify syntax
gh workflow list

# Check recent runs
gh run list --limit 5
```

### Tests Timing Out

**Issue:** Job exceeds 60 minutes

**Solution:**
1. Increase timeout (max 360 minutes)
2. Parallelized jobs (use matrix)
3. Skip slow tests in CI
4. Use workflow concurrency limits

```yaml
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true
```

### Artifacts Not Generated

**Check:**
1. Job succeeded or `continue-on-error: true`
2. Artifacts directory not empty
3. Retention not expired (30 days)

**Debug:**
```bash
# List step logs
gh run view {run_id} --log-failed
```

### API Key Issues

**Problem:** GEMINI_API_KEY not found

**Solution:**
1. Settings → Secrets and variables
2. Check `GEMINI_API_KEY` exists
3. Verify spelling exactly
4. Check repository vs organization secrets

### Costs and Usage

**Check GitHub Actions usage:**
1. Settings → Billing and plans
2. View Actions section
3. Shows minutes used

**Optimize:**
- Use scheduled triggers (less frequent)
- Cancel in-progress runs on new pushes
- Use matrix efficiently
- Skip unnecessary jobs

## Best Practices

### 1. Keep Workflows DRY

**Bad:**
```yaml
- run: python -m pytest
- run: python -m pytest
```

**Good:**
```yaml
- name: Run tests
  run: python -m pytest
```

### 2. Use Caching

```yaml
- uses: actions/setup-python@v5
  with:
    cache: 'pip'  # Cache pip dependencies
```

### 3. Fail Fast

```yaml
strategy:
  fail-fast: true  # Stop other jobs on first failure
```

### 4. Use Secrets Properly

```yaml
env:
  GEMINI_API_KEY: ${{ secrets.GEMINI_API_KEY }}
```

Never commit secrets to repo!

### 5. Document Workflows

```yaml
# Always add descriptions
name: Descriptive Name
jobs:
  my-job:
    name: Human-readable job name
```
